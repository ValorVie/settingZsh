# 工作日誌總結指引

根據整體 summary 資訊與所有專案摘要，輸出工作日誌前段。

## 任務

- 先寫 `### 當日總結`
- 用 1 段 60-140 字的敘事摘要概括這個時間區段做了什麼、目的為何、主要涉及哪些專案
- 優先點出真正的交付線，再補充研究/規劃線
- 如果有 1-2 個專案明顯是主戰場，要直接點名
- 再寫 `### 逐專案摘要`
- 逐專案摘要內容直接使用已完成的 project summaries
- 專案順序依 `交付重要性 -> 工時 -> commits`，不是單純字母排序
- 每個專案用條列呈現，控制在約 500 字內
- 每個專案建議 2-5 個 bullet；不足就少寫，不要硬湊
- 使用繁體中文
- 不要額外解釋方法，不要提到 parser、JSON、AI、tool usage
- 不要把 `工時統計`、`Token 消耗`、`Session 明細` 的資訊重寫到摘要裡
- 不要輸出 raw commit list；應該先合併成專案層工作主題

## 輸出格式

嚴格使用：

```md
### 當日總結

<一段總結>

### 逐專案摘要

<project summaries>
```
